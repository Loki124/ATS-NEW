## 1. 项目概述

### 1.1 项目背景
ATS (Applicant Tracking System) 招聘管理系统旨在为企业提供一套完整的招聘流程解决方案，涵盖从人才发现、简历筛选、面试评估到Offer发放、入职管理的全流程数字化管理。

### 1.2 项目目标
- 实现招聘流程的线上化管理，提高招聘效率
- 建立统一的人才数据库，实现人才资源复用
- 提供数据报表支持招聘决策
- 支持多角色协同工作（HR、HRBP、用人经理、面试官）

### 1.3 服务对象
| 角色 | 说明 |
|------|------|
| 超级管理员 | 系统最高权限，负责系统配置 |
| 管理员 | 业务管理员，负责日常运营 |
| HRBP | HR业务伙伴，对接业务部门 |
| HR | 招聘专员，执行招聘操作 |
| 用人经理 | 部门负责人，审批和面试 |
| 面试官 | 参与面试评估 |

## 2. 功能需求

## 3. 业务流程

### 3.1 招聘需求流程
```
需求创建 → 需求审批 → 需求通过
    ↓
职位发布 → 候选人推荐 → 简历筛选
    ↓
面试安排 → 面试评估 → 候选通过
    ↓
Offer发放 → 入职办理 → 招聘完成
```

### 3.2 候选人流程状态
1. 初评
2. HRBP筛选
3. 用人经理筛选
4. 用人经理上级筛选
5. 邀约
6. 联合面试
7. 综合面试
8. Offer沟通
9. 背调
10. 待入职
11. 入职

### 3.3 职位状态
- 招聘中
- 已停招
- 已完成

### 3.4 需求状态
- 草稿
- 进行中
- 已完成
- 已暂停
- 已停招

## 4. 非功能性需求

### 4.1 性能需求
- 页面加载时间 < 3秒
- API响应时间 < 1秒
- 支持100+并发用户

### 4.2 安全需求
- JWT Token认证
- 密码加密存储
- SQL注入防护
- XSS攻击防护

### 4.3 兼容性需求
- Chrome、Firefox、Safari、Edge最新两个版本
- 移动端适配（响应式设计）

## 5. 数据规范

### 5.1 需求编号规则
- 格式：`HC` + 6位数字
- 示例：HC000001、HC000012

### 5.2 薪资单位
- K：千元
- W：万元
- Y：元

### 5.3 状态枚举
- 启用/禁用
- 正常/锁定
- 招聘中/已停招/已完成
