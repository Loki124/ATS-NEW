## 跨计划契约（必须严格遵守）

1. **Candidate 状态扩展** (Plan A 独占): 11 个状态值在 schema 里定义, 后续 plan 引用这个枚举
2. **Permission V1+V2 现状**: 已有的 `permission.middleware.js` 不要改, Plan B 新建 `field-acl.middleware.js` 平行
3. **Service 命名约定**: `xxx.service.js` + `xxx.routes.js` + `xxx.test.js`, 全部 `src/services/__tests__/`
4. **前端 API 客户端**: `src/api/<domain>.ts` 单一文件, 不在 page 文件里直接 axios
5. **i18n 字段**: 中文硬编码（项目现状）, 不引入 i18n 库
6. **测试要求**: 每个新 service 至少 5 个单测, 状态机 / 关键算法 ≥ 10 个
7. **commit 规范**: `<type>(<scope>): <中文标题>`, 例 `feat(G44): 候选人 11 状态字段扩展`
8. **不破坏现有 266 个测试**: `npm test` 必须仍然 100% pass
